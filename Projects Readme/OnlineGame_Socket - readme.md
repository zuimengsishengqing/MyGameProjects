# Unity在线游戏项目OnlineGame_Socket

## 项目概述

本项目是一个基于Unity引擎开发的多人在线实时对战游戏，采用客户端-服务器架构。项目实现了完整的网络游戏功能，包括用户注册登录、房间管理、实时对战、位置同步、战斗系统等核心功能。项目使用Protocol Buffers进行数据序列化，通过TCP和UDP双协议实现网络通信，确保了关键数据的可靠传输和实时数据的高效传输。

### 主要特点

- 支持多人实时在线对战
- 完整的用户系统（注册、登录、房间管理）
- 实时位置同步和战斗系统
- 基于IPv6优先的网络连接方案
- 模块化的架构设计，易于扩展和维护

## 项目结构

### 整体架构

项目采用分层架构设计，主要包含以下层次：

- **表示层（UI层）**：负责用户界面显示和交互
- **业务逻辑层**：处理游戏逻辑和请求响应
- **数据访问层**：负责网络通信和数据序列化
- **网络传输层**：处理TCP/UDP底层通信

### 核心组件关系

**1. GameFace（游戏主控制器）**

- 单例模式，全局唯一实例
- 初始化并管理所有管理器
- 负责消息分发和全局协调

**2. Manager层（四大管理器）**

- **ClientManager**：网络通信管理，处理TCP/UDP连接、消息收发、域名解析
- **RequestManager**：请求管理，维护请求字典，分发响应到对应处理器
- **UIManager**：UI管理，使用栈结构管理面板层级，处理面板切换
- **PlayerManager**：玩家管理，处理玩家创建、销毁、位置同步、战斗逻辑

**3. Request层（请求处理系统）**

- BaseRequest基类定义请求生命周期
- 18个具体请求类处理不同业务逻辑
- 每个请求对应一个ActionCode，实现OnResponse方法处理响应

**4. UIPanel层（UI面板系统）**

- BasePanel基类定义面板生命周期（OnEnter、OnPause、OnRecovery、OnExit）
- 8个具体面板类实现不同界面功能
- 支持面板暂停和恢复机制

**5. Player层（玩家控制系统）**

- MyCharacterController：本地玩家输入处理（移动、跳跃）
- RemoteCharacter：远程玩家位置同步
- GunController：武器开火控制
- CharacterRistic：角色属性管理（血量、用户名等）

## 核心功能

### 1. 用户系统

- **用户注册**：支持新用户注册，验证用户名和密码
- **用户登录**：验证用户凭证，建立会话
- **会话管理**：维护用户登录状态

### 2. 房间系统

- **创建房间**：用户可创建新游戏房间
- **查找房间**：浏览可用房间列表
- **加入房间**：加入指定房间
- **退出房间**：离开当前房间
- **房间聊天**：房间内实时聊天功能

### 3. 游戏系统

- **开始游戏**：房主启动游戏
- **玩家同步**：实时同步所有玩家位置和状态
- **战斗系统**：支持开火、子弹击中、伤害计算
- **死亡判定**：血量归零时判定死亡
- **游戏结束**：显示游戏结果

### 4. 网络通信

- **TCP协议**：用于关键数据传输（登录、房间管理等）
- **UDP协议**：用于实时数据传输（位置更新、战斗数据）
- **IPv6优先**：优先使用IPv6连接，IPv4作为备用
- **域名解析**：支持通过域名动态解析服务器地址

### 5. 数据序列化

- **Protocol Buffers**：高效的数据序列化方案
- **消息封装**：TCP协议使用包头+包体结构，UDP协议直接传输包体
- **消息解析**：支持粘包处理和消息完整性验证

## 快速开始

### 环境要求

- Unity 2021.x 或更高版本
- Visual Studio 2019 或更高版本
- .NET Framework 4.x
- Google.Protobuf 库（已包含在项目中）

### 项目配置步骤

1. **导入项目**

   - 解压项目文件到本地目录
   - 使用Unity Hub打开项目

2. **网络配置**

   - 修改

     *ClientManager.cs*

     中的服务器地址：

     ```
     private const string domain = "gameconnection.dynv6.net";
     private const int tcpPort = 6666;
     private const int udpPort = 6667;
     ```

   - 确保服务器端已启动并开放相应端口

3. **场景配置**

   - 打开`SampleScene.unity`场景
   - 确保`GameFace`对象存在并挂载`GameFace.cs`脚本
   - 配置玩家生成点（SpawnPos）

4. **运行测试**

   - 点击Unity编辑器的Play按钮
   - 在登录界面输入用户名和密码
   - 创建或加入房间开始游戏

### 开发调试

1. **查看日志**

   - 使用Unity Console查看运行日志
   - 关键日志包括连接状态、请求发送、响应接收等

2. **网络测试**

   - 使用命令行工具测试网络连接：

     ```
     nslookup gameconnection.dynv6.net
     ```

   - 确认域名解析到正确的IPv6地址

3. **常见问题**

   - **连接失败**：检查网络连接、防火墙设置、服务器状态
   - **域名解析失败**：尝试使用IP地址替代域名
   - **数据同步异常**：检查Protocol Buffers版本兼容性

## 技术亮点

### 1. 双协议网络架构

- **TCP + UDP混合使用**：TCP保证可靠性，UDP保证实时性
- **智能协议选择**：关键操作使用TCP，实时数据使用UDP
- **连接超时检测**：实现5秒连接超时机制，避免长时间阻塞

### 2. 高效的消息序列化

- **Protocol Buffers**：比JSON/XML更高效，体积更小
- **自动代码生成**：OnlineGameSocket.cs由.proto文件自动生成
- **粘包处理**：Message类实现完整的消息缓冲和解析逻辑

### 3. 多线程任务调度

- **Loom类**：优雅地处理多线程和主线程之间的任务调度
- **异步任务执行**：后台线程执行耗时操作，主线程安全更新UI
- **延迟任务支持**：支持延迟执行和定时任务

### 4. 模块化架构设计

- **单一职责原则**：每个类职责明确，易于维护
- **依赖注入**：通过GameFace单例统一管理依赖
- **策略模式**：Request层使用策略模式处理不同请求
- **观察者模式**：UI面板使用栈管理，支持暂停和恢复

### 5. IPv6优先的网络方案

- **域名解析**：支持IPv6和IPv4双栈解析
- **自动降级**：IPv6不可用时自动降级到IPv4
- **动态配置**：通过dynv6服务实现动态域名解析

### 6. 完善的错误处理

- **异常捕获**：网络操作、消息解析等关键环节都有异常处理
- **状态检查**：Socket状态检查，避免无效操作
- **日志记录**：详细的日志输出，便于问题排查

### 7. 可扩展性设计

- **请求扩展**：新增请求只需继承BaseRequest并注册
- **面板扩展**：新增面板只需继承BasePanel并配置路径
- **协议扩展**：修改.proto文件即可扩展消息格式

### 8. 性能优化

- **对象池**：子弹等频繁创建销毁的对象可使用对象池
- **非阻塞接收**：UDP接收使用非阻塞模式，避免CPU占用过高
- **线程休眠**：接收线程适当休眠，平衡响应速度和资源消耗
